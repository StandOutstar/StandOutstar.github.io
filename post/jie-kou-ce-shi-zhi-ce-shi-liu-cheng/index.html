接口测试之-测试流程

2019-07-08

  1. 需求讨论

  2. 需求评审

  3. 场景设计

  4. 用例设计：

    1. 目的：
      • 理清思路，避免漏测；
      • 提高测试效率，方便后续测试；
      • 跟进测试进度；
      • 结果有记录；
      • 测试标准化
    2. 功能用例设计: 获取接口文档
    3. 逻辑用例设计: 是否依赖业务
    4. 异常用例设计
    • 参数异常:
      • 关键字参数：把语言关键字作为参数名，比如 echo
      • 参数多少：参数缺失，未知参数
    • 数据异常：
      • 关键字数据：把语言关键字作为参数值，比如 echo
      • 数据为空
      • 长度不一致
      • 错误数据
    1. 安全用例设计：
      • cookie
      • header
      • 唯一识别码：比如设备识别码

  5. 数据准备

  6. 执行
